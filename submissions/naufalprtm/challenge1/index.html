Challenge Details

This challenge focuses on creating a simple website with multiple HTML files that link to each other and deploying it on Arweave. The goal is to understand the basics of file-based routing, which is a foundational concept in web development. Here are some key points:

  • File-based Routing: Organize your website into separate HTML files and link them using anchor tags.
  • Deployment: Use tools like Dragondeploy to deploy your website to Arweave for permanent storage.
  • Purpose: Learn how to build static websites that fit Arweave's model and avoid server-side dependencies.

Challenge 1 Structure and Example Code

This challenge teaches the concept of file-based routing. In this structure, you create multiple HTML pages and link them together using anchor tags. Each page will represent a different part of your website, and the links between them will allow easy navigation.

Project Structure:

/project-directory
    /assets
        styles.css
        script.js
    index.html
    about.html
    tools.html

Example Code for index.html:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Frontend Challenge</title>
    <link rel="stylesheet" href="assets/styles.css">
</head>
<body>
    <header>
        <h1>Frontend Challenge 1</h1>
        <p>File-based Routing</p>
    </header>
    <nav>
        <a href="index.html">Home</a>
        <a href="about.html">Why?</a>
        <a href="tools.html">Tools</a>
    </nav>
    <main>
        <section>
            <h2>Welcome to the School of DumDum</h2>
            <p>Your one-stop journey to mastering Arweave AO development.</p>
        </section>
    </main>
    <footer>
        <p>© 2025 Frontend Challenge | Powered by Arweave</p>
    </footer>
</body>
</html>

Explanation:

The example above demonstrates the file structure and basic content of the homepage of your website. It uses anchor tags to navigate between three pages:

  • Home (index.html): The main page of the site.
  • Why? (about.html): A page explaining the concept of file-based routing.
  • Tools (tools.html): A page listing tools and resources for deployment.

Why File-based Routing?

File-based routing is one of the simplest ways to create websites. It avoids the complexity of additional tooling, making it ideal for beginners and small projects. On Arweave, where storage is permanent, this approach ensures that your website remains accessible forever without the need for server-side processing.
