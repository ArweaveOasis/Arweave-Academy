Why File-based Routing?

Overview

File-based routing is an essential method for creating simple, static websites. It aligns perfectly with Arweave's permanent storage model and ensures compatibility with AR.IO gateways that don't execute server-side code.

Why Choose This Challenge?

This challenge introduces developers to:

  • The fundamental concepts of Arweave and AO development.
  • Best practices for storing data and building applications on Arweave.
  • A hands-on experience to prepare for advanced development challenges.

Detailed Benefits

Adopting file-based routing offers several advantages:

  • Simplicity: No complex tooling is required. Developers can focus on building functional static websites with basic HTML, CSS, and JavaScript.
  • Permanent Storage: Arweave's unique model ensures that your website will be stored permanently, providing unmatched reliability and longevity.
  • Gateway Compatibility: As AR.IO gateways do not support server-side code execution, this method ensures smooth functionality across the network.
  • Cost-Effectiveness: Deploying a static website requires minimal resources, making it an affordable solution for individuals and small teams.

Real-World Use Cases

Here are some examples of where file-based routing can be effectively applied:

  • Personal Portfolios: Showcase your skills and projects with a lightweight, static website.
  • Documentation Sites: Create simple, accessible documentation for your projects or products.
  • Landing Pages: Deploy promotional or informational pages quickly and efficiently.

Conclusion

File-based routing is the perfect starting point for developers entering the Arweave ecosystem. It provides a foundation to build upon, enabling more complex development as skills progress.
